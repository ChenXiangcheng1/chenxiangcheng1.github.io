String，StringBuffer，StringBuilder的区别

String 字符串常量

  • final 不可变性，即不能进行字符串的修改和拼接 (因为String的不可变性能防止键改变，且重写了equals和hash方法，所以适合作为HashMap的键，Integer常量同理)
    每次对 String 类型进行修改的时候其实都等同于生成了一个新的 String 对象，然后将指针指向新的 String 对象，所以经常改变内容的字符串最好不要用 String 字符串常量引用
  • 线程安全，因为不可变性所以不会出现并发访问的问题

StringBuffer 线程安全的字符串变量

  • 可变
  • 线程安全

方法：

append(String): StringBuffer 同步非静态方法，附加拼接字符串

insert(int, char): StringBuffer 同步非静态方法，在指定的位置添加字符

indexOf(String): int 不包含str则返回-1，包含则对应返回下标

substring(s) 同步非静态方法，取下标 [s, ) 开始到末尾的字符串

substring(s,e) 同步非静态方法，取下标 [s,e) 的字符串

toString(): String 同步非静态方法，返回字符串

StringBuilder 非线程安全的字符串变量

  • 可变
  • 线程不安全

方法：

append(String): StringBuffer 附加拼接字符串

indexOf(String): int 不包含str则返回-1，包含则对应返回下标

equals 和 == 的区别

  • equals：在所有没有重写 equals() 方法的类中，调用equals()方法也是比较的地址值，然而在 Java 提供的所有类中，绝大多数类都重写了 equals() 方法，重写后的 equals() 方法一般都是比较两个对象的值

  • ==：比较的是两个对象的内存地址，即是否是引用相同的对象

所以如果两个对象，想让它们比较属性值的方式判断相等，就需要重载 equals() 和 hashcode() 方法

基本数据类型和包装类型的存储：

基本数据类型的存储
基本数据类型和stirng作为常量在方法区中的常量池以HashSet策略存储，即一个常量对应一个地址

基本数据的包装类型的存储
基本数据的包装类型是存储在堆内存，一个对象对应一个地址，对象的值可能相同

其他语言：
Python和C#中，== 是比较值
Java中，== 是比较地址
